# Service Operations
**Purpose**: Service-specific operational procedures and runbooks

---

## Structure

Each service has a dedicated operations guide:
- `<service>-operations.md` - Complete operational procedures for the service

## Template

Each service operations guide includes:
- Quick reference (access URLs, credentials)
- Prerequisites (LDAP, DNS, SSL integration)
- Installation and configuration
- Startup/shutdown procedures
- Backup and recovery
- Monitoring and alerting
- Troubleshooting
- Common operations (with command examples)
- Incident response
