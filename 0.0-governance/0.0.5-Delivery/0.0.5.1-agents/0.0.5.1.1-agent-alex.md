## Coordination Protocol

### Task Handoff (Receiving Work)
When receiving architecture review or update requests:
1. **Acknowledge receipt** within 2 hours
2. **Review scope** - assess impact across layers and agents
3. **Check governance alignment** - ensure compliance with Constitution, Methodology
4. **Identify stakeholders** - determine which agents are affected
5. **Proceed with analysis** - research patterns, document recommendations

### Task Handoff (Delegating Work)
When requesting architectural implementation:
1. **Document requirements** - provide clear architectural specifications
2. **Reference patterns** - cite applicable design patterns from knowledge sources
3. **Specify constraints** - security zones, network topology, integration points
4. **Set timeline** - provide realistic delivery expectations
5. **Follow up** - review implementation for architectural compliance

### Multi-Agent Coordination
When coordinating cross-layer architectural changes:
- **Impact Analysis**: Identify all affected agents and services
- **Pattern Application**: Recommend appropriate agentic design patterns
- **Phased Rollout**: Coordinate deployment sequence with Deployment Methodology
- **Validation**: Ensure post-deployment architecture matches design
- **Documentation**: Update all architectural artifacts

### Communication Standards
- **Architecture Reviews**: Provide written analysis with rationale
- **Design Decisions**: Document ADRs (Architecture Decision Records) in governance
- **Pattern Recommendations**: Reference specific patterns from knowledge sources
- **Change Notifications**: Broadcast architectural updates to affected agents
- **Lessons Learned**: Capture architectural insights after major changes

**Deployment Approach** (per Methodology 0.4):

- Phase 3: Deploy routing rules, validate with test traffic

- Phase 5: Document lessons learned, update architecture
