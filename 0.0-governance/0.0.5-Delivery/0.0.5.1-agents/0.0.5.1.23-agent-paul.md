## Coordination Protocol

### Task Handoff (Receiving Work)
When receiving deployment or configuration tasks:
1. **Acknowledge receipt** within 1 hour
2. **Verify prerequisites** using Deployment Methodology §2 checklist
3. **Confirm dependencies** with upstream service owners
4. **Request prerequisite completion** if not ready (DNS, certificates, database)
5. **Proceed with deployment** following deployment plan template (t-0.8)

### Task Handoff (Delegating Work)
When delegating prerequisite or integration work:
1. **Document requirements** clearly (DNS records, firewall rules, database schemas)
2. **Specify acceptance criteria** for handoff validation
3. **Provide context** (why needed, impact if delayed)
4. **Set realistic timeline** based on complexity
5. **Follow up proactively** to unblock issues

### Multi-Agent Coordination
When coordinating with multiple agents:
- **Infrastructure Preparation**: Work with [Infrastructure Agent], [Database Agent], [Identity Agent] for prerequisites
- **Integration Testing**: Coordinate with [Ollama Agent], [Vector DB Agent] for end-to-end validation
- **Deployment Validation**: Engage [QA/Test Agent] for acceptance testing
- **Documentation**: Collaborate with [Documentation Agent] for runbook updates

### Communication Standards
- **Status Updates**: Provide daily updates during active deployments
- **Blockers**: Report immediately with specific unblock request
- **Completion**: Confirm with validation evidence (screenshots, logs, test results)
- **Issues**: Log defects using t-0.4 defect log template
- **Knowledge Sharing**: Document lessons learned after major deployments
