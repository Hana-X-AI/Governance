## Example Interaction

**User (Service Owner)**: "I need to deploy the LiteLLM proxy service on hx-litellm-server."

**Agent** (using prompt): "I'll help configure hx-litellm-server for LiteLLM deployment. Here's the system-level setup:

**System Requirements Check**:
```bash
# Check Ubuntu version
lsb_release -a
# Ubuntu 24.04 LTS confirmed

# Check resources
free -h  # RAM: 32GB available
df -h    # Disk: 500GB available on /srv/litellm
nproc    # CPU: 16 cores
```

**Package Installation**:
```bash
# Update package lists
sudo apt update

# Install Python 3.12 and dependencies
sudo apt install -y python3.12 python3.12-venv python3-pip

# Install PostgreSQL client (for LiteLLM database)
sudo apt install -y postgresql-client

# Install Redis client
sudo apt install -y redis-tools
```

**User & Permissions**:
```bash
# Create litellm service user
sudo useradd -r -s /bin/bash -d /opt/litellm -m litellm

# Create storage directories
sudo mkdir -p /srv/litellm/{config,logs,cache}
sudo chown -R litellm:litellm /srv/litellm
sudo chmod 755 /srv/litellm
```

**Systemd Service Unit**:
```ini
# /etc/systemd/system/litellm.service
[Unit]
Description=LiteLLM Proxy Server
After=network.target postgresql.service redis.service
Wants=postgresql.service redis.service

[Service]
Type=simple
User=litellm
Group=litellm
WorkingDirectory=/opt/litellm
Environment="PATH=/opt/litellm/venv/bin"
ExecStart=/opt/litellm/venv/bin/litellm --config /srv/litellm/config/config.yaml
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

# Security hardening
NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/srv/litellm

[Install]
WantedBy=multi-user.target
```

**Firewall Configuration**:
```bash
# Allow LiteLLM port (4000)
sudo ufw allow 4000/tcp comment 'LiteLLM Proxy'

# Reload firewall
sudo ufw reload
```

**Performance Tuning**:
```bash
# Increase file descriptor limits for high concurrency
sudo tee /etc/security/limits.d/litellm.conf <<EOF
litellm soft nofile 65536
litellm hard nofile 65536
EOF

# Tune TCP settings for API server
sudo sysctl -w net.core.somaxconn=4096
sudo sysctl -w net.ipv4.tcp_max_syn_backlog=8192
```

**Service Management**:
```bash
# Reload systemd
sudo systemctl daemon-reload

# Enable service on boot
sudo systemctl enable litellm

# Start service
sudo systemctl start litellm

# Check status
sudo systemctl status litellm

# View logs
sudo journalctl -u litellm -f
```

**Monitoring Setup** (coordinate with Nathan Lewis):
```bash
# Install node_exporter for Prometheus metrics
sudo apt install -y prometheus-node-exporter
sudo systemctl enable prometheus-node-exporter
sudo systemctl start prometheus-node-exporter
```

**Coordination**:
- Maya Singh (LiteLLM Agent): Deploy application code and configuration
- Quinn Davis (Postgres): Ensure database connectivity
- Samuel Wilson (Redis): Verify cache connectivity
- Nathan Lewis (Metrics): Configure monitoring
- Isaac Morgan (CI/CD): Set up automated deployment

System is ready for LiteLLM deployment. Coordinate with Maya Singh to deploy the application."
