## Example Interaction

**User (Service Owner - Patricia Miller)**: "Help me containerize Ollama for deployment."

**Agent** (using prompt): "Based on Docker best practices, here's a comprehensive containerization strategy for Ollama:

**Dockerfile** (multi-stage build for optimization):
```dockerfile
# /srv/docker/dockerfiles/Dockerfile.ollama
FROM ubuntu:24.04 AS base

# Install dependencies
RUN apt-get update && apt-get install -y \\
    curl \\
    ca-certificates \\
    && rm -rf /var/lib/apt/lists/*

# Install Ollama
RUN curl -fsSL https://ollama.com/install.sh | sh

# Create non-root user for security
RUN useradd -r -s /bin/bash -u 1000 -d /home/ollama ollama \\
    && mkdir -p /home/ollama/.ollama \\
    && chown -R ollama:ollama /home/ollama

# Runtime stage
FROM base AS runtime

USER ollama
WORKDIR /home/ollama

# Expose Ollama API port
EXPOSE 11434

# Health check
HEALTHCHECK --interval=30s --timeout=10s --start-period=60s \\
  CMD curl -f http://localhost:11434/api/tags || exit 1

# Start Ollama server
CMD ["ollama", "serve"]
```

**Docker Compose** (with volumes and networks):
```yaml
# /srv/docker/compose/ollama/docker-compose.yml
version: '3.8'

services:
  ollama:
    build:
      context: ../../dockerfiles
      dockerfile: Dockerfile.ollama
    container_name: ollama-server
    hostname: hx-ollama-server
    ports:
      - "11434:11434"
    volumes:
      # Persistent model storage
      - ollama_models:/home/ollama/.ollama
    environment:
      - OLLAMA_HOST=0.0.0.0
      - OLLAMA_ORIGINS=*
    restart: unless-stopped
    deploy:
      resources:
        limits:
          cpus: '8'
          memory: 32G
        reservations:
          cpus: '4'
          memory: 16G
    networks:
      - hana_x_network
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

volumes:
  ollama_models:
    driver: local
    driver_opts:
      type: none
      o: bind
      device: /srv/ollama/models

networks:
  hana_x_network:
    name: hana_x
    driver: bridge
```

**Build Image**:
```bash
# Build Ollama image
docker build -f /srv/docker/dockerfiles/Dockerfile.ollama \\
  -t ollama:latest \\
  .

# Tag for versioning
docker tag ollama:latest ollama:1.0.0
```

**Deploy with Docker Compose**:
```bash
# Start Ollama container
cd /srv/docker/compose/ollama
docker-compose up -d

# Check status
docker-compose ps

# View logs
docker-compose logs -f ollama

# Pull a model
docker exec ollama-server ollama pull llama3.2
```

**Security Hardening**:
```dockerfile
# Add to Dockerfile for enhanced security
# Read-only root filesystem
--read-only \\
--tmpfs /tmp \\
--tmpfs /home/ollama/.ollama/tmp

# Drop capabilities
--cap-drop=ALL \\
--cap-add=NET_BIND_SERVICE

# Security options
--security-opt=no-new-privileges:true
```

**CI/CD Integration** (coordinate with Isaac Morgan):
```yaml
# .github/workflows/build-ollama.yml
name: Build Ollama Docker Image

on:
  push:
    paths:
      - 'docker/ollama/**'

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Build Docker image
        run: |
          docker build -f docker/ollama/Dockerfile \\
            -t ollama:${{ github.sha }} .

      - name: Push to registry
        run: |
          docker push ollama:${{ github.sha }}
```

**Monitoring** (coordinate with Nathan Lewis):
```bash

docker stats ollama-server

# Export metrics for Prometheus
docker run -d \\
  --name docker-exporter \\
  -p 9417:9417 \\
  -v /var/run/docker.sock:/var/run/docker.sock \\
  prometheuscommunity/docker-exporter
```
