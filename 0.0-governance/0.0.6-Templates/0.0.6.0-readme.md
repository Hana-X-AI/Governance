# Project Templates Directory
**Purpose**: Reusable templates for consistent project execution

---

## Overview

This directory contains standardized templates used throughout the Hana-X ecosystem for project planning, tracking, and deployment. All templates follow governance standards and provide consistent structure for documentation.

---

## Usage Guidelines

### For Project Managers
1. **Select appropriate template** based on work type
2. **Copy template** to project workspace
3. **Fill in all sections** following the template structure
4. **Update metadata** (dates, owners, versions)
5. **Reference related documents** for context

### For Agents
- Templates are referenced in agent profiles and work methodologies
- Use templates to maintain consistency across deliverables
- Update templates when patterns emerge or improvements identified

### Template Reference Pattern
```markdown
**Template Used**: `0.0.6.#-[name]-template.md`
```

---

## Template Standards

All templates in this directory:
- ✅ Follow hierarchical naming convention (`0.0.6.#-[name]-template.md`)
- ✅ Include standardized metadata headers
- ✅ Provide clear usage instructions
- ✅ Reference related governance documents
- ✅ Include version tracking in footers

---

## Adding New Templates

To add a new template:
1. **Create template file** following naming pattern: `0.0.6.##-[name]-template.md`
2. **Add metadata header** (Document Type, Created, Topic, Purpose, Classification)
3. **Include usage instructions** in template body
4. **Add standardized footer** with version tracking
5. **Update this README** with template description
6. **Cross-reference** related templates and governance docs
