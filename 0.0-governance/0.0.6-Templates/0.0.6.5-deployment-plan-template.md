### TASK-005: Systemd Service Creation
**Single Responsibility**: Create and enable systemd unit for service management

**Prerequisites**: TASK-003 (and TASK-004 if applicable) complete

**Agent Responsible**: [Service specialist]

**Coordination Required**: BB-8 (if complex systemd configuration)

**Implementation Steps**:
1. Create systemd unit file: `/etc/systemd/system/[service].service`
2. Reload systemd: `sudo systemctl daemon-reload`
3. Enable service: `sudo systemctl enable [service]`
4. Verify unit file: `systemctl cat [service]`

**Systemd Unit Template**:
```ini
[Unit]
Description=[Service Description]
After=network.target [dependencies].service
Requires=[dependencies].service

[Service]
Type=[simple/forking/notify]
User=[service-account]
Group=[service-group]
ExecStart=[/path/to/binary] [args]
Restart=on-failure
RestartSec=5s

[Install]
WantedBy=multi-user.target
```

**Validation**:
- [ ] Unit file created
- [ ] Systemd recognizes service: `systemctl status [service]`
- [ ] Service enabled for auto-start
- [ ] No syntax errors in unit file

**Exit Criteria**: Systemd service configured and enabled

**Rollback**:
```bash
sudo systemctl disable [service]
sudo rm /etc/systemd/system/[service].service
sudo systemctl daemon-reload
```

## 5. Rollback Procedure

*Per Deployment Methodology ¬ß5.5: Execute if deployment fails*

### 5.1 Rollback Triggers
Execute rollback if:
- Any critical task fails twice
- Service fails to start after configuration
- Integration tests reveal blocking issues
- Security validation fails
- Data corruption or loss risk identified

### 5.2 Rollback Steps
1. **Stop Service**: `sudo systemctl stop [service]`
2. **Disable Service**: `sudo systemctl disable [service]`
3. **Remove Systemd Unit**: `sudo rm /etc/systemd/system/[service].service && sudo systemctl daemon-reload`
4. **Remove Installation**: `rm -rf [installation path]`
5. **Remove Configuration**: `rm -rf /etc/[service]/`
6. **Remove Data** (if safe): `rm -rf /var/lib/[service]/` (CAUTION: Data loss)
7. **Update Platform Nodes**: Set status back to ‚¨ú TBD or üõ†Ô∏è In-Progress
8. **Document Issues**: Record what happened in deployment log

### 5.3 Post-Rollback Actions
- Root cause analysis before retry
- Update deployment plan if needed
- Consult with Agent Zero if unclear
